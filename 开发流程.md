# STM32F103C8T6 软件SPI读取W25Q64 — 开发流程

## 1. 项目概述

- **MCU**: STM32F103C8T6（Cortex-M3, 64KB Flash, 20KB SRAM）
- **外设**: W25Q64 SPI Flash（8MB），SSD1306 OLED（128x64, I2C）
- **通信方式**: 软件模拟SPI（GPIO位带操作）
- **构建工具**: Keil MDK uVision5 / EIDE (VSCode 插件)
- **调试器**: ST-Link
- **代码仓库**: [GitHub: rjsjing/stm32-spi-w25q64](https://github.com/rjsjing/stm32-spi-w25q64)

---

## 2. 开发环境搭建

### 2.1 安装 Keil MDK

1. 下载并安装 Keil MDK v5
2. 安装 STM32F10x 设备包（Pack Installer → STM32F1 Series）
3. 默认安装路径 `D:\keil5\`

### 2.2 安装 VSCode + EIDE 插件

1. 安装 **Embedded IDE (EIDE)** 插件: `cl.eide`
2. EIDE 会自动检测 Keil ARMCC 工具链（`ARMCC` / `AC5`）
3. 推荐同时安装：
   - `Cortex-Debug` (调试)
   - `ARM` 语法高亮
   - `Hex-Fmt` / `SRecord` (固件文件查看)

### 2.3 硬件连接

| STM32F103C8T6 | W25Q64 | OLED (SSD1306) |
|---------------|--------|-----------------|
| PB13 (引脚17) | SCK    | —               |
| PB14 (引脚18) | DO     | —               |
| PB15 (引脚19) | DI     | —               |
| PA4  (引脚10) | CS     | —               |
| PB5  (引脚25) | —      | SCL (I2C)       |
| PB6  (引脚26) | —      | SDA (I2C)       |
| 3.3V          | VCC    | VCC             |
| GND           | GND    | GND             |

### 2.4 打开项目

- **Keil**: 双击 `Project.uvprojx`，选择 "STM32F103C8" 作为目标设备
- **EIDE (VSCode)**: 用 VSCode 打开项目目录，EIDE 会自动导入项目

## 4. 开发流程

### 4.1 代码编写

```
推荐顺序（从底层到上层）：
  步骤1: MySPI.c/h       → 实现9个基础SPI时序函数（MySPI_Start、MySPI_Stop、MySPI_SwapByte...）
  步骤2: W25Q64.c/h      → 基于MySPI封装W25Q64命令（读ID、读写数据、擦除扇区）
  步骤3: OLED.c/h + Font → 显示驱动（字符、数字、字符串、进制输出）
  步骤4: Delay.c/h        → SysTick延时（微秒、毫秒、秒）
  步骤5: main.c           → 主逻辑：初始化→读取Flash→OLED显示
```

### 4.2 编译/构建

**Keil MDK**:
```
1. 打开 Project.uvprojx
2. 点击 "Project" → "Build Target" (F7)
3. 查看 Build Output 窗口，确认 0 Error(s), 0 Warning(s)
4. 产物: Objects/Project.axf, Objects/Project.hex
```

**EIDE (VSCode)**:
```
1. 按 Ctrl+Shift+P → "EIDE: Build"
2. 编译信息输出到终端
3. 产物: build/Target 1/Project.hex
```

### 4.3 烧录

**ST-Link 烧录步骤**:

1. 用 ST-Link 连接 PC 和开发板（SWDIO + SWCLK + GND）
2. 打开 Keil → Flash → Download (F8)
3. 或使用 STM32CubeProgrammer：
   ```
   STM32_Programmer_CLI.exe -c port=SWD -w build/Target 1/Project.hex
   ```
4. 烧录完成后按板载 RESET 键，程序开始运行

### 4.4 调试

**Keil 调试**:
```
1. Project → Options for Target → Debug → 选择 "ST-Link Debugger"
2. Port: SW
3. Debug → Start/Stop Debug Session (Ctrl+F5)
4. 设置断点 F9，运行 F5，单步 F10/F11
```

**VSCode Cortex-Debug 调试**:
```json
// launch.json 配置
{
    "type": "cortex-debug",
    "servertype": "stlink",
    "device": "STM32F103C8",
    "executable": "build/Target 1/Project.elf",
    "svdFile": "STM32F103.svd"
}
```

### 4.5 功能测试

1. 上电后 OLED 亮起，逐行显示 W25Q64 信息
2. 预期显示：`MANUFACTURER ID` / `DEVICE ID` / 读出的数据
3. 用逻辑分析仪或示波器测量 SPI 引脚波形（可选）

## 5. 编码规范

### 5.1 GPIO 配置

全部使用 `PB` 端口（PB5/PB6 I2C），`PB13/PB14/PB15` SPI + `PA4` CS：

```c
// 初始化示例 (MySPI.c)
#define W25Q64_SCK_PIN   GPIO_Pin_13    // PB13
#define W25Q64_MISO_PIN  GPIO_Pin_14    // PB14
#define W25Q64_MOSI_PIN  GPIO_Pin_15    // PB15
#define W25Q64_CS_PIN    GPIO_Pin_4     // PA4
```

### 5.2 函数命名

- 驱动层: `芯片名_操作()`，例如 `MySPI_Init()`、`MySPI_SwapByte()`
- 用户层: 英文小写下划线，例如 `w25q64_read_id()`
- 硬件初始化: 放在 `main.c` 中的 `while(1)` 之前

### 5.3 时钟配置

- HSE = 8MHz → PLL 9倍频 → SYSCLK = 72MHz
- APB1 = 36MHz, APB2 = 72MHz
- 由 `system_stm32f10x.c` 中的 `SystemInit()` 自动完成

## 7. 常见问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| OLED无显示 | I2C时序过慢，设备未应答 | 增加延时或检查PB5/PB6接线 |
| W25Q64读回全0xFF | CS引脚未拉低或SPI时序错误 | 检查PA4是否拉低，确认模式0/3正确 |
| Keil编译报错未定义引用 | `stm32f10x_conf.h` 中对应外设未 `#include` | 确认 `USE_STDPERIPH_DRIVER` 已定义 |
| ST-Link 连接失败 | 接线错误或驱动未安装 | 检查SWDIO/SWCLK/GND，重装ST-Link驱动 |
| 烧录成功但不运行 | Flash起始地址不正确 | 确保 `ROM1: 0x8000000, 0x10000`（Keil Target Options） |
| Github推送被拒 | 代理未配置 | `git config --global http.proxy http://127.0.0.1:<port>` |
